1.Client — Server
    Client (trình duyệt, app) gửi HTTP request tới server.
    Server (ngược lại) xử lý rồi trả HTTP response gồm status code, headers, body.
2.Chu trình cơ bản
    Client gõ URL → browser kiểm tra cache → nếu cần resolves domain → gửi request → server trả response → browser hiển thị.
3.DNS (Domain Name System)
    Chuyển tên miền (ví dụ example.com) thành IP (ví dụ 93.184.216.34).
    Có nhiều loại bản ghi: A (IPv4), AAAA (IPv6), CNAME, MX, TXT.
4.Địa chỉ IP & cổng (port)
    IP: định danh thiết bị trên mạng.
    Port: “cổng” dịch vụ trên thiết bị (HTTP thường dùng 80, HTTPS dùng 443, dev hay dùng 5000/8080/8888…).
    127.0.0.1 = localhost (máy mình). 0.0.0.0 = lắng nghe mọi interface.
5.URL / URI — thành phần
    scheme://host:port/path?query#fragment
    ví dụ: https://example.com:443/users?id=5#bio
    Scheme: http/https — xác định giao thức và có ảnh hưởng đến port mặc định.
6.HTTP Request (các phần chính)
    Method: GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS.
    Path + Query string.
    Headers: Host, User-Agent, Accept, Content-Type, Authorization, Cookie...
    Body: dữ liệu gửi (thường ở POST/PUT).
7.HTTP Response
    Status code (mã lỗi/chỉ báo).
    Response headers: Content-Type, Set-Cookie, Cache-Control, Location...
    Body: HTML, JSON, file, v.v.
8.TLS / HTTPS
    HTTPS = HTTP qua TLS/SSL → kết nối mã hoá.
    Certificat e (SSL) đảm bảo server là chính xác và truyền bảo mật.
9.Cookie & Session
    Cookie: dữ liệu server gửi lưu trên client; dùng xác thực, lưu trạng thái.
    Session: dữ liệu phía server liên kết với cookie ID.
10.Cache & CDN
    Cache: giảm tải, trả lại nội dung đã lưu.
    CDN: phân phối nội dung qua nhiều node toàn cầu → giảm độ trễ.
11.Reverse Proxy,Load Balancer, Gateway
    Reverse proxy (Nginx) nhận request, chuyển tới backend (Gunicorn, uWSGI, Node).
    Load balancer phân phối tải giữa nhiều server.
    Gateway / API gateway xử lý routing, auth, rate limiting.
12.Các logs & nơi debug
    Browser DevTools (Network tab) — xem request/response, headers, thời gian.
    Server logs (/var/log/..., logs ứng dụng).
    Reverse proxy logs (Nginx), process manager logs (systemd, pm2).
    Error/stack trace để tìm nguyên nhân 500.
13. IP (Internet Protocol)
    Là địa chỉ định danh thiết bị trong mạng.
    Giống như “số nhà” để gửi dữ liệu đúng nơi.
    Mỗi thiết bị (máy tính, router, server, điện thoại...) có một địa chỉ IP.
    Có 2 phiên bản: IPv4 và IPv6.
14.IPv4 vs IPv6
Đặc điểm	              IPv4         	         IPv6
Độ dài	                    32 bit	           128 bit
Dạng hiển thị               4 nhóm số (0–255)      8 nhóm hex, ví dụ 2001:0db8:85a3::8a2e:0370:7334
Số địa chỉ tối đa           ~4.3 tỷ	           3.4 × 10³⁸ (rất lớn, gần như vô hạn)
Cạn kiệt	            Đã gần hết	           Gần như không giới hạn
Bảo mật	                    Không tích hợp sẵn	   Tích hợp IPsec (bảo mật tốt hơn)
Cấu hình	            Dễ dàng, hỗ trợ rộng   Khó hơn, chưa phổ biến toàn cầu
Broadcast (gửi toàn mạng)	Có	           Không có (thay bằng multicast)
15.Subnet (Mạng con)
Subnet = sub-network = “mạng con”.
Dùng để chia nhỏ mạng lớn thành nhiều mạng nhỏ.
Mỗi subnet có địa chỉ riêng, giúp quản lý và giới hạn truy cập.
Ví dụ:
Mạng tổng: 192.168.0.0/24
/24 nghĩa là 24 bit đầu là mạng, 8 bit sau là host.
→ Có thể chứa 256 địa chỉ (0–255), trừ 2 địa chỉ đặc biệt:
.0 (địa chỉ mạng)
.255 (broadcast)
Thực tế sử dụng được 254 thiết bị (từ .1 đến .254).
=> Subnet mask tương ứng: 255.255.255.0
Nó dùng để xác định phần “mạng” và phần “host” trong IP.
16.Gateway (Cổng mặc định)
Gateway là thiết bị trung gian giúp máy tính ra ngoài mạng cục bộ (LAN) đến Internet hoặc mạng khác.
Thường là router của bạn.
Ví dụ:
Máy bạn: 192.168.1.10
Subnet mask: 255.255.255.0
Default gateway: 192.168.1.1 (router WiFi)
→ Khi bạn truy cập Google, máy gửi gói tin qua gateway này.
Tóm tắt:
Trong mạng nội bộ: gói tin chuyển thẳng.
Ra ngoài mạng: phải đi qua gateway.
17.NAT (Network Address Translation)
NAT = dịch địa chỉ mạng
Giúp nhiều thiết bị dùng chung một IP công cộng.
Ví dụ:
Mạng nhà bạn có:
 Laptop: 192.168.1.2
 Điện thoại: 192.168.1.3
 Router: IP riêng (WAN) là 113.22.35.120
Khi truy cập mạng:
 Router dịch IP nội bộ → IP công cộng.
 Internet chỉ thấy IP 113.22.35.120 (của router).
Có 3 loại NAT:
Loại	                                                 Mô tả
Static NAT	                                Ánh xạ 1–1 giữa IP nội và IP ngoài.
Dynamic NAT	                                IP ngoài được chọn ngẫu nhiên trong danh sách có sẵn.
PAT / Overload (Port Address Translation)	Nhiều IP nội dùng chung một IP ngoài, phân biệt bằng số cổng (port). Đây là kiểu dùng trong WiFi gia đình.
18.Cách các thành phần này hoạt động cùng nhau
Ví dụ khi bạn mở https://example.com từ laptop:
 Laptop có IP nội: 192.168.1.5
 Gửi yêu cầu ra ngoài → router (gateway: 192.168.1.1)
 Router dùng NAT để dịch sang IP công cộng 113.22.35.120
 Gói tin đi qua ISP → DNS tra IP thật của example.com
 Dữ liệu trả về router → NAT dịch ngược → máy bạn nhận.
